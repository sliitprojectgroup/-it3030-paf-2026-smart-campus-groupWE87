# QR-Based Check-In Feature Implementation Plan

## Overview
Implement a simple URL-based QR check-in: when a booking is approved, a QR code is generated. Scanning it opens `/verify/{qrCode}` which calls the backend to mark the booking as checked-in.

---

## Backend Changes (4 files)

### 1. `Booking.java` â€” Add 2 new fields
- Add `qrCode` (String, unique, nullable) field
- Add `checkedInTime` (LocalDateTime) field
- `checkedIn` already exists (Boolean), no change needed

### 2. `BookingRepository.java` â€” Add finder method
- Add `Optional<Booking> findByQrCode(String qrCode);`

### 3. `BookingService.java` â€” Modify approve + add verify
- In `approveBooking()`: set `booking.setQrCode(UUID.randomUUID().toString())` before save
- Add `verifyBooking(String qrCode)` method:
  - Find booking by qrCode â†’ throw ResourceNotFoundException if not found
  - If status != APPROVED â†’ throw ConflictException("Booking is not approved")
  - If checkedIn == true â†’ return booking (already checked in)
  - Else: set checkedIn=true, checkedInTime=LocalDateTime.now(), save, return booking

### 4. `BookingController.java` â€” Add verify endpoint
- Add `GET /api/bookings/verify/{qrCode}` that calls `bookingService.verifyBooking(qrCode)`
- Returns the Booking entity on success

---

## Frontend Changes (4 files + 1 new file + 1 npm install)

### 5. Install dependency
- `npm install qrcode.react` in frontend/

### 6. `api.js` â€” Add verify API call
- Add `export const verifyBooking = (qrCode) => api.get(\`/bookings/verify/${qrCode}\`);`

### 7. `MyBookings.jsx` â€” Add QR modal for APPROVED bookings
- Add state: `qrBooking` (null or booking object) to control modal
- For APPROVED bookings, add "View QR" button in the Actions column
- Modal shows `<QRCodeSVG value={\`${window.location.origin}/verify/${booking.qrCode}\`} />` plus booking details and check-in status
- Close button to dismiss modal

### 8. `VerifyBooking.jsx` â€” New page (create)
- Extract `qrCode` from URL params via `useParams()`
- On mount, call `verifyBooking(qrCode)` API
- Show loading â†’ then success/error result
- Simple centered card UI: green checkmark on success, red X on error
- This page must be OUTSIDE the Layout/ProtectedRoute so phone scanning works without auth

### 9. `App.jsx` â€” Add verify route
- Add route: `<Route path="/verify/:qrCode" element={<VerifyBooking />} />`
- Place it OUTSIDE the `<Route path="/" element={<ProtectedRoute><Layout /></ProtectedRoute>}>` block, at the same level as `/login`

## Testing Instructions

1. Start backend: `mvn spring-boot:run`
2. Start frontend: `npm run dev -- --host`
3. Find your local IP (e.g. `192.168.x.x`)
4. Create a booking â†’ status = PENDING
5. Admin approves â†’ `qrCode` is generated in DB
6. Open My Bookings â†’ click "View QR" on the approved booking
7. Scan QR with phone camera â†’ opens `http://<IP>:5173/verify/<qrCode>`
8. Verify page calls API â†’ booking marked as checked-in
9. Scan again â†’ shows "Already checked in" message
